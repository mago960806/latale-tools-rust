## 字节序规范

### 字节序定义

**所有 LaTale 资源文件格式均使用 Little-Endian 字节序。**

### 多字节数值存储

| 数据类型 | 字节大小 | 字节序 | 存储格式 |
|---------|---------|--------|----------|
| int16/short | 2 bytes | Little-Endian | 低字节在前，高字节在后 |
| int32/long | 4 bytes | Little-Endian | 0x01 02 03 04 表示 0x04030201 |
| int64 | 8 bytes | Little-Endian | 低到高字节依次存储 |
| float (IEEE 754) | 4 bytes | Little-Endian | 符合 IEEE 754 标准的位布局 + Little-Endian |

### 读取规则

读取多字节数值时：
1. 按字节顺序从低地址到高地址读取
2. 将读取的字节按 Little-Endian 规则组装成数值
3. 示例：读取 4 字节 `01 00 00 00` → 组装为 0x00000001

## 数据对齐规则

### 结构体对齐

**大多数结构体使用 4 字节对齐。**

### 基础数据类型大小

| 数据类型 | 字节大小 | 对齐要求 |
|---------|---------|----------|
| bool | 1 byte | 1 byte |
| int8/char | 1 byte | 1 byte |
| int16/short | 2 bytes | 2 bytes |
| int32/long | 4 bytes | 4 bytes |
| int64/long long | 8 bytes | 8 bytes |
| float | 4 bytes | 4 bytes |
| 指针 | 4 bytes (32-bit) | 4 bytes |

### 布尔值填充规则

- bool 字段占用 1 字节
- 后续非 bool 字段可能需要填充到 4 字节边界
- 填充字节值未定义，读取时应跳过

### 示例结构体布局

```
偏移量  类型              字段名
0x00    int               Value1          (4 bytes)
0x04    float             Value2          (4 bytes)
0x08    bool              Flag             (1 byte)
0x09    byte[3]           Padding          (3 bytes - 填充)
0x0C    int               Value3          (4 bytes)
总计：   12 bytes
```

## 字符串存储格式

### 固定长度字符串

#### LDT_STR64 (64字节字符串)

```
偏移量  类型      字段名
0x00    char[64]  String    64字节C风格字符串
```

**格式规范：**
- 固定长度字符数组
- 以 null 结尾 ('\0')
- 未使用字节填充为 0
- 读取时需找到第一个 null 终止符确定实际长度

#### 路径字符串 (255/64字节)

```
偏移量  类型      字段名
0x00    char[255] Path      255字节路径字符串
0x00    char[64]  Name       64字节名称字符串
```

**格式规范：**
- 固定长度缓冲区
- C 风格 null 终结
- 可能包含相对路径或绝对路径

#### 大字符串

```
偏移量  类型      字段名
0x00    int       Length    字符串长度 (不含null)
0x04    char[]    Data      可变长度字符串数据
0x04+Length  char      Null      null 终止符
```

**格式规范：**
- 先存储 4 字节长度值
- 后跟字符数据
- 最后以 null 终止
- 最大长度限制：8192 字节

### 字符串编码

- 所有字符串使用 **ASCII** 或 **GBK** 编码
- 路径分隔符：`/` 或 `\\`
- 文件名区分大小写

## 文件路径结构

### 资源目录结构

```
DATA/
├── ANITABLE/          # 动画表文件 (.SEQ)
├── BGFORMAT/          # 背景格式文件 (.BGI, .TXT)
├── TERRAIN/           # 地形文件 (.ATTR, .FORM, .PAL)
├── GLOBALTERRAIN/     # 全局地形文件
├── CHAR/              # 角色资源
│   ├── CHARLAYER/    # 角色图层
│   └── MONSTER/       # 怪物/NPC资源
├── FX/                # 特效资源 (.FXG, .FXM)
├── LDT/               # 数据库文件 (.LDT)
├── INTERFACE/         # UI资源
├── BACKGROUND/        # 背景图片
├── BGM/               # 背景音乐
├── CURSOR/            # 光标资源
└── LOADING/           # 加载画面
```

### 文件扩展名定义

| 扩展名 | 类型 | 描述 | 是否支持二进制 |
|-------|------|------|---------------|
| .SEQ | 动画序列 | 角色动画数据 | 是 |
| .MOB | 怪物动画 | 怪物/NPC动画数据 | 是 |
| .ARG | 攻击数据 | 战斗攻击信息 | 是 |
| .DRG | 受击数据 | 战斗受击信息 | 是 |
| .BGI | 背景信息 | 背景图像配置 | 是 |
| .PAL | 调色板 | 地形调色板 | 是 |
| .FORM | 地形形状 | 地形形状数据 | 是 |
| .ATTR | 地形属性 | 地形属性数据 | 是 |
| .FXG | 特效组 | 特效组定义 | 是 |
| .FXM | 特效模型 | 特效模型数据 | 是 |
| .STG | 关卡数据 | 关卡/地图数据 | 是 |
| .LDT | 数据库 | 游戏数据库 | 否 |
| .TXT | 文本 | 配置文本文件 | 否 |
| .TGA/.PNG | 图片 | 纹理/图片资源 | 否 |

## 版本控制机制

### 版本号定义

| 文件类型 | 版本字段位置 | 当前版本 | 版本格式 |
|---------|-------------|----------|----------|
| 动画表 (.SEQ) | 第一个 float | 2.0 | 浮点数 |
| 背景格式 (.BGI) | 第一个 float | 0.1 | 浮点数 |
| 地形调色板 (.PAL) | 第一个 float | 0.1 | 浮点数 |
| 关卡数据 (.STG) | 第一个 float | 0.4 | 浮点数 |
| 特效模型 (.FXM) | 第一个 float | 3.2 | 浮点数 |
| LDT数据库 | 文件头中 | 1.0 | 整数 |

### 版本检测流程

1. **读取版本号**
   - 从文件开头读取一个 float (4 bytes)
   - 或者读取一个 int (4 bytes) 并转换为 float

2. **版本比较**
   - 如果读取的版本号 > 当前支持版本号 → 不支持的文件格式
   - 如果读取的版本号 == 当前支持版本号 → 使用完整格式
   - 如果读取的版本号 < 当前支持版本号 → 根据版本号选择读取逻辑

3. **字段存在性判断**
   - 通过版本号判断某些字段是否存在
   - 新版本字段在旧版本文件中不存在时使用默认值

### 特效系统版本历史

| 版本 | 新增字段 | 兼容性变更 |
|------|---------|-------------|
| 1.0 → 1.1 | Loop, ScreenRender, iFollowParent | 向后兼容 |
| 1.1 → 2.0 | bool bLightMap → TEXTURE_EFFECT eTextureEffect | 不兼容 |
| 2.0 → 2.1 | fFrameScale 分离为 fFrameScaleX/Y | 向后兼容 |
| 2.1 → 3.0 | 粒子系统重构 | 不兼容 |
| 3.0 → 3.1 | uiFXModelID, eFXModelType | 向后兼容 |
| 3.1 → 3.2 | iFXMRenderLayer | 向后兼容 |

## 文件格式标识

### 二进制文件标识

#### 动画/特效文件标识

```
值: 415
类型: int
位置: 文件第二个字段
用途: 验证文件为有效的二进制格式文件
```

#### ASCII 文件标识

```
格式: TEXT_FORMAT
位置: 文件开头第一行
用途: 标识文件为文本格式而非二进制
```

### 文件头结构

#### 通用文件头布局

```
偏移量  类型          字段名            描述
0x00    int/float     Version          版本号
0x04    int           Identifier1       格式标识1 (可选)
0x08    int           Identifier2       格式标识2 (可选)
0x0C    ...           ...               其他头信息
```
