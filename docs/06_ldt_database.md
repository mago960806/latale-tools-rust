## LDT 文件格式

### 文件结构概述

LDT 文件是一种自定义的二进制数据库格式，由以下主要部分组成：

1. **文件头** - 固定 8708 字节，包含数据库元数据和字段定义
2. **数据区** - 存储所有数据行
3. **索引区** - 主键索引和字段名索引（内存中构建）

### 数据库文件命名

```
DATA/LDT/GLOBAL_RES.LDT
DATA/LDT/GLOBAL_STRING.LDT
DATA/LDT/ITEM.LDT
DATA/LDT/MOBLOOT.LDT
DATA/LDT/FX_LIST.LDT
DATA/LDT/NPC.LDT
...
```

## 字段类型定义

### 8种字段类型

| 类型代码 | 类型名称 | 值域 | 存储大小 | 描述 |
|---------|---------|------|----------|------|
| 0 | fldNA | - | 0 bytes | 无效字段 |
| 1 | fldString | 字符串 | 变长 | 字符串，最大 8192 字节 |
| 2 | fldTF | true/false | 4 bytes | 布尔值（存储为 long） |
| 3 | fldNum | 整数 | 4 bytes | 有符号整数 (long) |
| 4 | fldPer | 百分比 | 4 bytes | 浮点数 (float) |
| 5 | fldFID | 外键 ID | 4 bytes | 关联其他表的 ID |
| 6 | fldAlias | 字符串 | 变长 | 别名，最大 4096 字节 |
| 7 | fldNum64 | 大整数 | 8 bytes | 64 位有符号整数 |
| 255 | fldPrimaryKey | 主键 | - | 主键标识 |

### 字段类型特征说明

- **fldNA**: 无效字段，占位符
- **fldString**: 变长字符串，前 4 字节存储长度，后跟字符串数据
- **fldTF**: 布尔值，以 4 字节整数存储，0 为 false，非 0 为 true
- **fldNum**: 标准 32 位有符号整数
- **fldPer**: 单精度浮点数，用于百分比等数值
- **fldFID**: 外键引用，用于关联其他数据表
- **fldAlias**: 变长字符串，格式同 fldString，但最大 4096 字节
- **fldNum64**: 64 位有符号整数，用于大数值
- **fldPrimaryKey**: 主键标识，用于唯一标识数据行

## 文件头布局

### 文件头结构

文件头固定 8708 字节，布局如下：

| 偏移量 | 类型 | 字段名 | 大小 | 描述 |
|--------|------|--------|------|------|
| 0x00 | long | nDB_ID | 4 bytes | 数据库 ID |
| 0x04 | long | nFields | 4 bytes | 字段数量 (最多 128 个) |
| 0x08 | long | nData | 4 bytes | 数据行数 |
| 0x0C | LDT_STR64[128] | FieldNAM | 8192 bytes | 字段名称数组 (每个 64 字节) |
| 0x200C | long[128] | FieldTYP | 512 bytes | 字段类型数组 (每个 4 字节) |
| **总计** | - | - | **8708 bytes** | |

### LDT_STR64 字符串类型

```text
struct LDT_STR64 {
    char String[64];  // 64 字节固定长度字符串
};
```

- 用于存储字段名称
- 固定 64 字节长度
- 以 NULL 结尾
- 未使用的字节填充为 0

## 字段数据存储

### 数据行结构

每行数据由以下部分组成：

1. **主键** (8 bytes) - 64 位整数，唯一标识该行
2. **字段数据** - 按字段顺序存储所有字段值

### 各类型存储格式

#### fldString (字符串类型)

```
+--------+--------+--------+--------+-----------------+
| 长度 (4 bytes)      | 字符串数据 (变长)   |
+--------+--------+--------+--------+-----------------+
```

- 前 4 字节：字符串长度（ little-endian long）
- 后续字节：实际字符串数据
- 最大长度：8192 字节

#### fldTF (布尔类型)

```
+--------+--------+--------+--------+
| 值 (4 bytes, long)      |
+--------+--------+--------+--------+
```

- 4 字节整数
- 0 = false
- 非 0 = true

#### fldNum (整数类型)

```
+--------+--------+--------+--------+
| 值 (4 bytes, long)      |
+--------+--------+--------+--------+
```

- 标准 32 位有符号整数
- little-endian 字节序

#### fldPer (百分比类型)

```
+--------+--------+--------+--------+
| 值 (4 bytes, float)     |
+--------+--------+--------+--------+
```

- 单精度浮点数 (IEEE 754)
- little-endian 字节序
- 用于百分比、倍率等数值

#### fldFID (外键 ID)

```
+--------+--------+--------+--------+
| ID (4 bytes, long)      |
+--------+--------+--------+--------+
```

- 4 字节整数
- 用于关联其他数据表

#### fldAlias (别名类型)

```
+--------+--------+--------+--------+-----------------+
| 长度 (4 bytes)      | 字符串数据 (变长)   |
+--------+--------+--------+--------+-----------------+
```

- 格式同 fldString
- 前 4 字节：字符串长度
- 后续字节：实际字符串数据
- 最大长度：4096 字节

#### fldNum64 (64位整数)

```
+--------+--------+--------+--------+--------+--------+--------+--------+
| 值 (8 bytes, __int64)                                      |
+--------+--------+--------+--------+--------+--------+--------+--------+
```

- 64 位有符号整数
- little-endian 字节序
- 用于大数值或主键

### 数据行示例

假设一个表有 3 个字段：fldNum, fldString, fldPer

```
行数据布局：
+------------------+------------------+------------------+------------------+
| 主键 (8 bytes)    | 字段0 (4 bytes)  | 字段1 (变长)     | 字段2 (4 bytes)  |
+------------------+------------------+------------------+------------------+
```

## 索引机制

### 主键索引

- **索引类型**: 哈希表 (HashMap)
- **键**: 主键值 (64 位整数)
- **值**: 数据行在线性数组中的索引
- **用途**: 通过主键快速定位数据行

**工作原理**：
1. 读取数据时，为每行数据的主键创建索引条目
2. 主键 -> 数据行索引的映射
3. 查询时通过主键直接定位到目标行

### 字段名索引

- **索引类型**: 哈希表 (HashMap)
- **键**: 字段名称字符串
- **值**: 字段序号 (0-127)
- **用途**: 通过字段名快速定位字段索引

**工作原理**：
1. 读取文件头时，为每个字段名创建索引条目
2. 字段名 -> 字段序号的映射
3. 查询时通过字段名直接定位到字段索引

### 线性数据库

- 所有字段数据按行优先顺序存储在连续数组中
- 每个字段占用一个数组元素
- 主键索引用于快速定位行的起始位置

### 查询流程

1. **通过主键查询**:
   - 使用主键索引查找行索引
   - 定位到行的起始位置
   - 读取所需字段

2. **通过字段名查询**:
   - 使用字段名索引查找字段序号
   - 定位到目标字段
   - 读取字段值

## 二进制格式布局

### 完整文件布局

```
+--------------------------+
| 文件头 (8708 bytes)       |
+--------------------------+
| 数据行 0                  |
| - 主键 (8 bytes)         |
| - 字段 0                 |
| - 字段 1                 |
| - ...                    |
+--------------------------+
| 数据行 1                  |
| - 主键 (8 bytes)         |
| - 字段 0                 |
| - 字段 1                 |
| - ...                    |
+--------------------------+
| ...                      |
+--------------------------+
| 数据行 N-1                |
| - 主键 (8 bytes)         |
| - 字段 0                 |
| - 字段 1                 |
| - ...                    |
+--------------------------+
```

### 文件头详细布局

```
偏移量    大小    类型            字段名            描述
0x0000    4       long            nDB_ID           数据库 ID
0x0004    4       long            nFields          字段数量
0x0008    4       long            nData            数据行数
0x000C    64      LDT_STR64       FieldNAM[0]      第 1 个字段名称
0x004C    64      LDT_STR64       FieldNAM[1]      第 2 个字段名称
...      ...      ...             ...              ...
0x1FEC    64      LDT_STR64       FieldNAM[127]    第 128 个字段名称
0x200C    4       long            FieldTYP[0]      第 1 个字段类型
0x2010    4       long            FieldTYP[1]      第 2 个字段类型
...      ...      ...             ...              ...
0x2204    4       long            FieldTYP[127]    第 128 个字段类型
```

### 数据类型参考

| 类型名称 | 大小 | 字节序 | 说明 |
|---------|------|--------|------|
| long | 4 bytes | Little-Endian | 32 位有符号整数 |
| __int64 | 8 bytes | Little-Endian | 64 位有符号整数 |
| float | 4 bytes | IEEE 754 | 单精度浮点数 |
| LDT_STR64 | 64 bytes | - | 固定长度字符串 |

## 字符串类型定义

### LDT_STR64

```
大小: 64 字节
用途: 存储字段名称
结构: char String[64]
特性: 固定长度，NULL 结尾
```

### 字符串编码

- **编码格式**: 可能是 ASCII 或 GBK（中文支持）
- **最大长度**:
  - fldString: 8192 字节
  - fldAlias: 4096 字节
  - LDT_STR64: 63 字节（64 字节缓冲区）
- **终止符**: NULL (0x00)
