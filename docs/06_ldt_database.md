## LDT 文件格式

### 文件结构概述

LDT 文件是一种自定义的二进制数据库格式，由以下主要部分组成：

1. **文件头** - 固定 8716 字节，包含数据库元数据和字段定义
2. **数据区** - 存储所有数据行

### 数据库文件命名

```
DATA/LDT/GLOBAL_RES.LDT
DATA/LDT/GLOBAL_STRING.LDT
DATA/LDT/ITEM.LDT
DATA/LDT/MOBLOOT.LDT
DATA/LDT/FX_LIST.LDT
DATA/LDT/NPC.LDT
...
```

## 字段类型定义

### 8种字段类型

| 类型代码 | 类型名称 | 值域 | 存储大小 | 描述 |
|---------|---------|------|----------|------|
| 0 | fldNA | - | 0 bytes | 无效字段 |
| 1 | fldString | 字符串 | 变长 | 字符串，最大 65535 字节 |
| 2 | fldTF | true/false | 4 bytes | 布尔值（存储为 long） |
| 3 | fldNum | 整数 | 4 bytes | 有符号整数 (long) |
| 4 | fldPer | 百分比 | 4 bytes | 浮点数 (float) |
| 5 | fldFID | 外键 ID | 变长 | 关联其他表的 ID（格式: "spf_id,row_id"） |
| 6 | fldAlias | 字符串 | 变长 | 别名，最大 65535 字节 |
| 7 | fldNum64 | 大整数 | 8 bytes | 64 位有符号整数 |

### 字段类型特征说明

- **fldNA**: 无效字段，占位符，不占用存储空间
- **fldString**: 变长字符串，前 2 字节存储长度（little-endian u16），后跟字符串数据（GBK 编码）
- **fldTF**: 布尔值，以 4 字节整数存储，0 为 false，非 0 为 true
- **fldNum**: 标准 32 位有符号整数
- **fldPer**: 单精度浮点数，用于百分比等数值
- **fldFID**: 外键引用，存储格式同 fldString，内容为 "spf_id,row_id"
- **fldAlias**: 变长字符串，格式同 fldString
- **fldNum64**: 64 位有符号整数，用于大数值

## 文件头布局

### 文件头结构

文件头固定 8716 字节，布局如下：

| 偏移量 | 类型 | 字段名 | 大小 | 描述 |
|--------|------|--------|------|------|
| 0x00 | long | nDB_ID | 4 bytes | 数据库 ID |
| 0x04 | long | nFields | 4 bytes | 字段数量 (最多 128 个) |
| 0x08 | long | nData | 4 bytes | 数据行数 |
| 0x0C | LDT_STR64[128] | FieldNAM | 8192 bytes | 字段名称数组 (每个 64 字节) |
| 0x200C | long[128] | FieldTYP | 512 bytes | 字段类型数组 (每个 4 字节) |
| **总计** | - | - | **8716 bytes** | |

计算：4 + 4 + 4 + (128 × 64) + (128 × 4) = 12 + 8192 + 512 = 8716 bytes

### LDT_STR64 字符串类型

```text
struct LDT_STR64 {
    char String[64];  // 64 字节固定长度字符串
};
```

- 用于存储字段名称
- 固定 64 字节长度
- 以 NULL (0x00) 结尾
- NULL 之后的剩余字节填充为空格 (0x20)
- **空字段**: 全部填充为 0x00

**示例：** 字段名 `_Type`(5 字符) 的存储方式：

```
5F 54 79 70 65 00 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20
_  T  y  p  e  \0 [--------空格填充到64字节---------]
```

**空字段**的存储方式：

```
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 ... (64 bytes all zeros)
```

## 字段数据存储

### 数据行结构

每行数据由以下部分组成：

1. **主键** (4 bytes) - 32 位整数，唯一标识该行
2. **字段数据** - 按字段顺序存储所有字段值

### 各类型存储格式

#### fldString / fldAlias / fldFID (变长字符串类型)

```
+--------+--------+-----------------+
| 长度 (2 bytes, u16) | 字符串数据 (变长)   |
+--------+--------+-----------------+
```

- 前 2 字节：字符串长度（little-endian u16）
- 后续字节：实际字符串数据（GBK 编码）
- 最大长度：65535 字节

#### fldTF (布尔类型)

```
+--------+--------+--------+--------+
| 值 (4 bytes, long)      |
+--------+--------+--------+--------+
```

- 4 字节整数
- 0 = false
- 非 0 = true

#### fldNum (整数类型)

```
+--------+--------+--------+--------+
| 值 (4 bytes, long)      |
+--------+--------+--------+--------+
```

- 标准 32 位有符号整数
- little-endian 字节序

#### fldPer (百分比类型)

```
+--------+--------+--------+--------+
| 值 (4 bytes, float)     |
+--------+--------+--------+--------+
```

- 单精度浮点数 (IEEE 754)
- little-endian 字节序
- 用于百分比、倍率等数值

#### fldNum64 (64位整数)

```
+--------+--------+--------+--------+--------+--------+--------+--------+
| 值 (8 bytes, __int64)                                      |
+--------+--------+--------+--------+--------+--------+--------+--------+
```

- 64 位有符号整数
- little-endian 字节序
- 用于大数值

### 数据行示例

假设一个表有 3 个字段：fldNum, fldString, fldPer

```
行数据布局：
+------------------+------------------+------------------+------------------+
| 主键 (4 bytes)   | 字段0 (4 bytes)  | 字段1 (变长)     | 字段2 (4 bytes)  |
+------------------+------------------+------------------+------------------+
```

## 二进制格式布局

### 完整文件布局

```
+--------------------------+
| 文件头 (8716 bytes)       |
+--------------------------+
| 数据行 0                  |
| - 主键 (4 bytes)         |
| - 字段 0                 |
| - 字段 1                 |
| - ...                    |
+--------------------------+
| 数据行 1                  |
| - 主键 (4 bytes)         |
| - 字段 0                 |
| - 字段 1                 |
| - ...                    |
+--------------------------+
| ...                      |
+--------------------------+
| 数据行 N-1                |
| - 主键 (4 bytes)         |
| - 字段 0                 |
| - 字段 1                 |
| - ...                    |
+--------------------------+
| 文件尾 (64 bytes)         |
| - "END" (3 bytes)        |
| - 空格填充 (61 bytes)     |
+--------------------------+
```

### 文件尾 (Footer)

每个LDT文件末尾有一个固定的64字节footer：

```
45 4E 44 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20 20
E  N  D  [-----------61个空格(0x20)填充-----------]
```

### 文件头详细布局

```
偏移量    大小    类型            字段名            描述
0x0000    4       long            nDB_ID           数据库 ID
0x0004    4       long            nFields          字段数量
0x0008    4       long            nData            数据行数
0x000C    64      LDT_STR64       FieldNAM[0]      第 1 个字段名称
0x004C    64      LDT_STR64       FieldNAM[1]      第 2 个字段名称
...      ...      ...             ...              ...
0x1FEC    64      LDT_STR64       FieldNAM[127]    第 128 个字段名称
0x200C    4       long            FieldTYP[0]      第 1 个字段类型
0x2010    4       long            FieldTYP[1]      第 2 个字段类型
...      ...      ...             ...              ...
0x2204    4       long            FieldTYP[127]    第 128 个字段类型
0x220C    -       -               数据区开始        第 1 行数据
```

### 数据类型参考

| 类型名称 | 大小 | 字节序 | 说明 |
|---------|------|--------|------|
| long | 4 bytes | Little-Endian | 32 位有符号整数 |
| __int64 | 8 bytes | Little-Endian | 64 位有符号整数 |
| float | 4 bytes | IEEE 754 | 单精度浮点数 |
| u16 | 2 bytes | Little-Endian | 16 位无符号整数（字符串长度） |
| LDT_STR64 | 64 bytes | - | 固定长度字符串 |

## 字符串编码

### 数据存储编码

- **编码格式**: GBK（中文支持）
- **字符串长度**: 2 字节无符号整数（最大 65535 字节）
- **字段名称**: ASCII/GBK，固定 64 字节缓冲区

### CSV 导出编码

CSV 导出时字符串会被转换为 UTF-8 编码，以便于编辑和查看。
